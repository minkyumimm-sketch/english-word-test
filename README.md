# 英単語テスト作成アプリ

Excel、またはGoogleスプレッドシートで管理している英単語データから、条件を指定して
英単語テスト（問題・解答）を自動生成し、ブラウザでそのまま印刷できるアプリです。

このドキュメントは、**利用者（先生・生徒）向けの使い方**と、**開発・保守を行う人向けの技術情報**の
両方をまとめたものです。目的に応じて該当する見出しを読んでください。

## 1. アプリ概要

### 何をするアプリか

Excel、またはGoogleスプレッドシートに登録した英単語データ（順位・英語・日本語訳など）を
ブラウザに読み込み、

- 出題する順位範囲・問題数・出題方向（英→日／日→英）・出題順（順位順／ランダム）
- 1回分だけのテスト、または複数日分をまとめて自動生成

といった条件を指定するだけで、**問題ページ**と**解答ページ**をその場で作成し、
ブラウザの印刷機能でそのまま紙に印刷（またはPDF保存）できるツールです。

- Excel／Googleスプレッドシート = 単語データの管理専用（先生が普段どおり編集するだけ）
- Webアプリ = 問題生成・表示・印刷専用

という役割分担にしており、ExcelマクロやVBAの知識がなくても運用できます。
どちらのデータ形式を使うかは自由に選べ、後から切り替えることもできます
（[4. データの作り方](#4-データの作り方excelgoogleスプレッドシート) 参照）。

### 対象利用者

| 利用者 | 用途 |
|---|---|
| 先生 | Excel（`Data/` フォルダ等）またはGoogleスプレッドシートで単語データを管理し、授業・宿題用のテストを作成する |
| 生徒 | ブラウザでテストを生成し、自習用に印刷する（先生が許可した範囲で） |

ログイン機能・生徒個人の成績管理はありません（同じPC・同じブラウザを複数人で共有する
教室運用を想定した、シンプルなツールです）。

## 3. インストール方法

起動方法は3通りあります。**普段使いには方法A（公開URLを開く）が最も手軽**です。
社内ネットワークのみで使いたい場合やオフライン環境では、方法B・方法Cを使ってください。
いずれの方法で開いても、アプリの機能・見た目は同じです。

### 方法A: 公開URLを開く（一番簡単・インストール不要）

GitHub Pagesで公開されている場合は、以下のURLをブラウザで開くだけで使えます。
ダウンロードや設置作業は一切不要です。

```
https://<GitHubユーザー名>.github.io/<リポジトリ名>/
```

※ 上記はURLの形式です。実際のURLは公開設定後に確定します
（公開手順は [12. GitHub Pagesでの公開（管理者向け）](#12-github-pagesでの公開管理者向け) を参照）。

### 方法B: ダブルクリックで開く（ローカル・オフライン）

ネットに繋がない環境で使いたい場合や、開発・動作確認をしたい場合の方法です。

1. このプロジェクトのフォルダ一式を、そのままPC内の好きな場所にコピーします
   （Desktopでも共有フォルダでも構いません）。フォルダの中身（`index.html`、`css/`、
   `js/`、`lib/`、`Data/` など）の相対的な位置関係を変えなければ、特別なセットアップは
   不要です。
2. `index.html` をダブルクリックします。
3. ブラウザ（Chrome推奨）でアプリの画面が開きます。

### 方法C: ローカルサーバーを使用する場合の起動方法

環境によっては `file://` で直接開くと一部のブラウザ機能が制限される場合があります。
Node.js がインストールされていれば、簡易サーバー経由で起動することもできます。

```bash
# プロジェクトのルートフォルダで実行
node server.js
```

起動後、ブラウザで次のURLを開きます。

```
http://localhost:5500/
```

ポート番号を変更したい場合は、環境変数 `PORT` を指定します。

```bash
# Windows PowerShellの例（ポート8080で起動）
$env:PORT=8080; node server.js
```

## 4. データの作り方（Excel／Googleスプレッドシート）

単語データはExcel・Googleスプレッドシートのどちらでも作成できます。**表の形式は完全に共通**
（同じ列判定ロジックを共有）なので、まずは共通ルールを読んだうえで、使いたい方式の
節（4.2または4.3）を読んでください。

### 4.1 共通のデータ形式

1行目を**ヘッダー行**、2行目以降を**データ行**として読み込みます（Excel・Googleスプレッドシート共通）。

#### 必須列・任意列

| 列 | 必須/任意 | 説明 |
|---|---|---|
| 順位 | **必須** | 出題範囲の絞り込みに使う数値。数値として読み取れない行はスキップされます。 |
| 英語 | **必須** | 英単語。空欄の行はスキップされます。 |
| 訳（日本語） | **必須** | 日本語の意味。空欄の行はスキップされます。 |
| レベル | 任意 | 「レベル指定」の絞り込みに使う文字列（例:「最重要レベル」）。空欄でも読み込めます。 |
| 掲載ページ | 任意 | 参考情報として保持しますが、テスト画面には表示されません。 |

#### 列名（ヘッダーの付け方）

列の並び順は自由です。**1行目のセルの文字列に、以下のキーワードが含まれているか**で
列の役割を自動判定します。

| 役割 | ヘッダーに含める文字列（いずれか） |
|---|---|
| レベル | `レベル` |
| 順位 | `順位` |
| 掲載ページ | `ページ` |
| 英語 | `英語` または `英単語` |
| 訳 | `訳` または `日本語` または `意味` |

キーワードに一致する列が見つからない場合は、既定の並び（**A列=レベル、B列=順位、
C列=掲載ページ、D列=英語、E列=訳**）として読み込まれます。列名を多少アレンジしても
（例:「英単語」「日本語訳」など）自動判定されるため、既存のExcel資産をそのまま使えます。

#### データ入力時の注意点

- 1行目は必ずヘッダー行にしてください（データはヘッダー行に関わらず2行目から読み込まれます）。
- 「順位」は整数にしてください。小数・文字列・空欄の行は自動的にスキップされます
  （エラーにはなりませんが、その行はテストに出題されません）。
- 「英語」または「訳」が空欄の行もスキップされます。
- 同じ順位の重複や、順位の欠番があっても読み込み自体は可能です（並び替えて処理されます）。

#### サンプル

```
| レベル       | 順位 | 掲載ページ | 英語    | 訳       |
|--------------|------|------------|---------|----------|
| 最重要レベル | 1    | p26-p27    | the     | その     |
| 最重要レベル | 2    | p26-p27    | to      | ～に     |
| 最重要レベル | 3    | p26-p27    | I       | 私は     |
| 標準レベル   | 401  | p120-p121  | enjoy   | ～を楽しむ |
```

`Data/` フォルダに実際のサンプル（先生用の単語データExcel）が入っているので、
列構成を確認したい場合はそちらも参考にしてください。

### 4.2 Excelを使う場合

#### 対応形式

| 拡張子 | 対応状況 |
|---|---|
| `.xlsx` | 対応 |
| `.xlsm` | 対応（マクロ部分は読み込み時に無視されます） |
| `.xls` | ファイル選択自体は可能ですが、動作未検証です |

#### Excel固有の注意点

- シート名は `Sheet1` を優先して読み込みます。無い場合は `単語` `単語データ` `データ` の
  いずれか、それも無ければ一番左のシートを読み込みます。
- アプリ画面の「ファイルを選択」または画面へのドラッグ＆ドロップで読み込みます。

### 4.3 Googleスプレッドシートを使う場合

Excelの代わりに、Googleスプレッドシートで単語データを管理することもできます。
**複数の先生・複数のPCから同じデータを共同編集したい場合**や、**GitHub Pages公開版で
ファイルの受け渡しをせずにデータを更新したい場合**に向いています。

#### 公開方法（スプレッドシート側の設定）

1. Googleスプレッドシートを開きます。
2. メニューから **ファイル → 共有 → ウェブに公開** を選びます。
3. 公開する範囲（シート全体、または特定のシート）を選び、**公開** を押します。
   確認ダイアログが出た場合は「OK」を押して公開を確定します。
4. 表示されたURL、または再度スプレッドシートを開いて共有設定を確認したときのURLを
   コピーします（後述の「URL取得方法」参照）。

※「共有」ボタンで「リンクを知っている全員が閲覧可」に設定しただけでは、
「ウェブに公開」とは異なります。CSVとして安定して取得するためには、上記の
「ウェブに公開」の手順を行ってください。

#### URL取得方法

アプリの「GoogleスプレッドシートのURL、または公開CSVのURL」欄には、次のいずれかを
貼り付けられます。

| 貼り付けるURLの種類 | 例 |
|---|---|
| 通常の編集画面のURL（ブラウザのアドレスバーのURL） | `https://docs.google.com/spreadsheets/d/xxxxx/edit#gid=0` |
| 「ウェブに公開」で発行されたURL | `https://docs.google.com/spreadsheets/d/e/xxxxx/pubhtml` |
| 既にCSV形式で取得できるURL | `https://docs.google.com/spreadsheets/d/xxxxx/export?format=csv` |

どの形式で貼り付けても、アプリ側で自動的にCSV取得用のURLへ変換します
（入力欄の下に変換後のURLがプレビュー表示されます）。特定のシート（タブ）を
指定したい場合は、URLに含まれる `gid=` の数値がそのまま引き継がれます。

#### 更新方法

1. Googleスプレッドシート側でデータを編集し、保存します（Googleスプレッドシートは
   自動保存のため、特別な保存操作は不要です）。
2. アプリ画面に戻り、「読み込む」ボタンをもう一度押します。

Excelと同様、**編集内容は自動では反映されません**。「読み込む」を押した時点の
最新データが取得されます。

#### Excelとの違い

| 項目 | Excel | Googleスプレッドシート |
|---|---|---|
| データの受け渡し | ファイルをコピー・共有する必要がある | URLを共有するだけでよい（ファイルの受け渡し不要） |
| 複数人での編集 | 同時編集はしづらい | Googleアカウントがあれば共同編集しやすい |
| 読み込み操作 | ファイルを選択（毎回ファイル自体を選ぶ） | URLを一度入力すれば、以後は「読み込む」ボタンのみでよい |
| 通信 | 不要（ローカルファイルを読むだけ） | 必要（スプレッドシートの取得に通信する） |
| GitHub Pages公開版との相性 | Excelファイルを別途配布する必要がある | URLさえ共有すれば、公開されたアプリからそのまま読み込める |
| オフライン利用 | 可能 | 不可（読み込み時のみ通信が必要。一度読み込めばオフラインでも生成・印刷は可能） |

## 5. 使用方法

### 5.1 データ読込

1. アプリ画面の「1. 単語データの読み込み」で、「データ読込方法」から
   **Excelファイル** または **Googleスプレッドシート** を選びます
   （データ形式の作り方は [4. データの作り方](#4-データの作り方excelgoogleスプレッドシート) を参照）。
   - Excelファイルの場合: 「ファイルを選択」ボタンを押すか、Excelファイルを画面に
     ドラッグ＆ドロップします。
   - Googleスプレッドシートの場合: URL欄にスプレッドシートのURL（または公開CSVのURL）を
     貼り付け、「読み込む」ボタンを押します。
2. 読み込みが成功すると、読み込んだ件数（例:「2000件の単語を読み込みました」）が表示されます。
3. 一度読み込んだデータはブラウザに保存され、次回アプリを開いたときに自動で復元されます
   （詳細は [7. データ保存](#7-データ保存localstorage) を参照）。データ元を更新した場合は、
   その都度ファイルを選び直す／「読み込む」を押し直してください（自動では更新を検知しません）。
4. 読込方法は後からいつでも切り替えられます。切り替えても、既に読み込み済みのデータは
   新しいデータを読み込むまで保持されます。

#### 現在の教材表示

データ読み込み欄のすぐ下に「現在の教材」という小さなカードが常に表示されます。
ここには、**実際にテスト生成に使われている（最後に読み込みに成功した）教材**の名前と
読込方法（Excelファイル／Googleスプレッドシート）が表示され、教材を読み込む・
切り替えるたびに即座に更新されます。ラジオボタンを切り替えただけ（まだ読み込んでいない）
状態では変わらず、実際に読込が成功した時点で更新される点に注意してください。
「どの教材で今テストを作ろうとしているか」を勘違いしたまま印刷してしまうミスを防ぐための
表示です。

| 状態 | 表示例 |
|---|---|
| 何も読み込んでいない | 教材未読込 |
| Excel読込後 | 📄 高校英単語.xlsx／読込方法：Excelファイル |
| Googleスプレッドシート読込後 | ☁️ 中学英単語／読込方法：Googleスプレッドシート |

Googleスプレッドシートの教材名は、可能であれば実際のスプレッドシートのタイトルを
自動取得して表示します（「ウェブに公開」されたURLの場合のみ、取得は数百ミリ秒ほど
遅れて表示が更新されることがあります）。取得できない場合はURLの末尾から分かる範囲の
名称、それも難しい場合は「Googleスプレッドシート」という表示になります。

この教材名は、単語データと同じくブラウザに保存され、次回起動時も自動で復元されます
（詳細は [7. データ保存](#7-データ保存localstorage) を参照）。

### 5.2 条件設定

「2. テスト条件の設定」で、以下を指定します。

| 項目 | 内容 |
|---|---|
| タイトル（任意） | 印刷ページの見出しに使う文字列。空欄の場合は「英単語テスト」になります。 |
| 開始順位 | 出題する順位の開始位置（単発・複数日分どちらのモードでも使用） |
| 終了順位 | 出題する順位の終了位置（**単発モードのみ**。複数日分モードでは代わりに「作成日数」を使います） |
| レベル指定 | Excelの「レベル」列に基づく絞り込み（空欄=すべて対象） |
| 出題方向 | 英語→日本語 ／ 日本語→英語 |
| 出題順 | 順位順 ／ ランダム |
| 生成モード | 単発（1回分のテスト） ／ 複数日分を一括生成 |

### 5.3 単発テスト

生成モードで「単発（1回分のテスト）」を選んだ場合、追加で「問題数」を指定します。
指定した順位範囲・レベルに合致する単語の中から、指定した問題数だけ出題されます
（該当する単語が問題数より少ない場合は、該当する分だけ出題されます）。

### 5.4 一括テスト

生成モードで「複数日分を一括生成」を選んだ場合、**終了順位の代わりに**以下を指定します
（開始順位は単発モードと共通です）。

| 項目 | 内容 |
|---|---|
| 1日あたりの順位幅 | 順位範囲をこの幅で区切り、1区切り＝1日分のテストにします |
| 作成日数 | 何日分のテストをまとめて作るか |
| 1日あたりの問題数 | 各日の区切りの中から、何問出題するか |

例えば「開始順位1・順位幅100・作成日数5・1日あたり問題数20」と指定すると、
「1日目（1〜100位）」「2日目（101〜200位）」「3日目（201〜300位）」
「4日目（301〜400位）」「5日目（401〜500位）」の5セットがまとめて生成されます。

#### 生成予定プレビュー

条件を入力すると、「テストを生成する」ボタンの上に**生成予定**が自動で表示され、
実際に生成する前に日程ごとの範囲・問題数と、合計の対象語数・総出題数を確認できます。
開始順位・順位幅・作成日数・1日あたりの問題数・レベル指定を変更すると、その場で
再計算されます。

#### 単語数が範囲の途中で尽きる場合

指定した範囲・日数に対して単語データが不足する場合（多くは最終日）は、エラーには
ならず、実際に存在する順位までを自動的に対象とします。生成予定にも、例えば
「5日目は順位401〜450が対象です」のように、範囲が縮んだことが分かるよう表示されます。
単語データが完全に尽きた日（対象0件）は、その日だけ自動的にスキップされます
（それより前の日番号がずれることはありません）。

### 5.5 印刷

1. 「テストを生成する」を押すと、「3. プレビュー・印刷」にプレビューが表示されます。
2. 一括生成の場合は、日程ごとのタブで切り替えて確認できます。
3. 「問題プレビュー／解答プレビュー」で見た目を確認できます。
4. 「印刷対象」で、印刷する内容を選びます。

   | 選択肢 | 内容 |
   |---|---|
   | 問題のみ | 生徒に配るテスト用紙 |
   | 解答のみ | 先生用の解答一覧 |
   | 問題→解答（全日程） | 各日程ごとに「問題→解答」の順で連続印刷（自己採点用など） |

5. 「印刷する」を押すとブラウザの印刷ダイアログが開きます。複数日分の場合は、
   1日ごとに自動で改ページされます。

文字サイズ・行間・列数（1列/2列）・ページの余白は、問題数や単語・訳の長さに応じて
自動調整されます。手動での調整は不要です。

さらに、問題ページ・解答ページはそれぞれ個別に「1ページに収まるか」を判定し、
はみ出す場合は自動で調整します。まず文字を縮めずに済む方法（問題ページの見出し・
名前欄・点数欄を印刷時だけ1行にまとめる、丸括弧書きの補足説明を印刷時だけ少し
小さく表示する、1列レイアウトを2列に戻せないか試す）を優先し、それでも収まらない
場合にのみ余白・行間隔・文字サイズ等を必要最小限だけ
縮小します。読みやすさを保てる範囲でのみ調整するため、問題数や訳文が非常に長い
場合は無理に縮小せず2ページ以上になることがあります。複数日分をまとめて生成した
場合も、各日・各ページごとに個別に最適なレイアウトが計算されます。

## 6. 印刷について

### 推奨設定

- 用紙サイズ: **A4縦**（アプリ側でA4縦を前提にレイアウトを計算しています）
- 余白: ブラウザの印刷ダイアログでは「**デフォルト（標準）**」を選んでください
  （アプリ側で用紙の余白込みのレイアウトを計算しているため、「余白なし」にすると
  内容が用紙端に寄りすぎる場合があります）。
- 拡大/縮小: 「**なし（100%）**」を推奨します。
- 背景のグラフィック: オフのままで問題ありません（背景色を使用していないため）。

### ブラウザ印刷方法

1. アプリ画面で「印刷する」ボタンを押します（またはブラウザの印刷ショートカット
   `Ctrl+P` でも印刷ダイアログを開けます）。
2. 印刷ダイアログで「送信先（プリンター）」を選びます。
3. 上記の推奨設定（用紙: A4、余白: デフォルト、倍率: 100%）を確認します。
4. 「印刷」を押します。

### PDF保存方法（ブラウザ標準機能）

追加のソフトは不要です。印刷ダイアログの「送信先」で
**「PDFに保存」**（Chromeの場合。Edgeでは「Microsoft Print to PDF」）を選び、
保存先を指定して保存するだけで、印刷内容がPDFファイルとして保存されます。

## 7. データ保存（localStorage）

このアプリは、ブラウザの **localStorage**（そのブラウザ・そのPCの中だけに保存される
仕組み）を使って、次回起動時の入力の手間を減らしています。外部サーバーへの送信は
一切行われません。この仕組みは、GitHub Pagesで公開したURLから使う場合も、
`file://` で直接開いた場合も、`node server.js` で起動した場合も同じように動作します
（保存場所は「開いたURL（オリジン）ごと」のため、同じURLで開けば復元されます）。

| 保存されるキー | 内容 | 保存されるタイミング |
|---|---|---|
| `wordTestApp.lastWords.v1` | 最後に読み込んだ単語データ・レベル一覧・データ元の名称（「現在の教材」表示に使用） | Excel／Googleスプレッドシートの読み込み成功時（Googleスプレッドシートのタイトルが後から取得できた場合は、その時点でも再保存） |
| `wordTestApp.lastSettings.v1` | 最後に使用したテスト条件一式（タイトル・順位範囲・問題数・出題方向・出題順・レベル指定・生成モード・1日あたりの順位幅・作成日数・1日あたりの問題数・印刷対象 など） | テスト生成成功時、および印刷対象の変更時 |
| `wordTestApp.lastLoadMethod.v1` | 最後に使用したデータ読込方式（Excel／Googleスプレッドシート）と、入力したGoogleスプレッドシートのURL | Excel読込成功時、Googleスプレッドシート読込成功時 |

### リセット方法

| やりたいこと | 方法 |
|---|---|
| 読み込んだ単語データだけ消したい | アプリ画面の「読み込んだデータを消去」ボタンを押す |
| テスト条件（前回の設定）だけ消したい／全部リセットしたい | 現時点ではアプリ内にボタンはありません。ブラウザの開発者ツールを開き（`F12`）、コンソールタブで次のコマンドを実行してください: `localStorage.clear()` |

「読み込んだデータを消去」ボタンは単語データのみを消去し、テスト条件（前回設定）は
残ります。Excelを読み込み直すたびに条件を入力し直さずに済むようにするための仕様です。

## 9. よくある質問（FAQ）

**Q. Excelを編集したら、アプリ側も自動で最新の内容になりますか？**
A. なりません。Excelを編集・保存した後は、アプリ画面で改めてファイルを選び直して
読み込んでください（自動監視はしていません）。

**Q. 入力したExcelデータやテスト条件は、インターネット上のどこかに送信されますか？**
A. されません。すべてブラウザ内（あなたのPCの中）だけで処理され、外部サーバーとの
通信は一切行いません。

**Q. 「現在の教材」がずっと「Googleスプレッドシート」のままで、シート名になりません。**
A. スプレッドシートのタイトルは、「ウェブに公開」されたURLの場合にのみ自動取得を試みます。
それでも取得できない場合（未公開、通信環境、Google側の仕様変更等）は、URLから読み取れる
名称、それも無理なら既定の「Googleスプレッドシート」という表示のままになります。動作に
支障はありませんが、シート名で見分けたい場合は「ウェブに公開」の設定を確認してください。

**Q. スマートフォンやタブレットでも使えますか？**
A. 画面は表示できますが、印刷・条件入力はPC・ノートPCでの利用を想定して作られています。
モバイル向けの最適化は行っていません。

**Q. 生徒に問題用紙だけ配りたいのですが？**
A. 印刷対象で「問題のみ」を選んで印刷してください。先生の手元用に解答が欲しい場合は、
別途「解答のみ」で印刷するか、「問題→解答（全日程）」でまとめて印刷してください。

**Q. 複数のExcelファイル・スプレッドシート（学年ごとなど）を切り替えて使えますか？**
A. 使えます。ファイルを選び直す・別のURLを読み込むたびに、その内容に読み込みが
置き換わります（保存されるのは常に「最後に読み込んだ1件」です）。

**Q. 4択問題や穴埋め問題は作れますか？**
A. 現時点では対応していません（単語を見て意味を書く・意味を見て単語を書く、という
記述式のみです）。

**Q. Excelで使っていたデータを、そのままGoogleスプレッドシートに移行できますか？**
A. できます。列の形式（順位・英語・訳・レベル・掲載ページ）が共通のため、Excelの内容を
コピーしてGoogleスプレッドシートに貼り付け、「ウェブに公開」するだけで使えます。

**Q. Googleスプレッドシートを読み込むと、内容がGoogleや外部に送信されますか？**
A. アプリからは、あなたが指定したスプレッドシートのURLへ直接読み込みにいくだけです
（ブラウザからGoogleのサーバーへCSVを取得しにいく通信のみ発生します）。読み込んだ後の
データや、テスト条件・生成結果が、Google以外の外部サーバーに送信されることはありません。

保守・拡張を行う場合は、以下のドキュメントもあわせて参照してください。

- [docs/DESIGN.md](docs/DESIGN.md) — データモデル、各モジュールの責務、

- [CLAUDE.md](CLAUDE.md) — 開発方針・技術スタックの決定事項・作業ルール。

### 技術スタックの要点

- ビルドツール不要の静的HTML/CSS/JS（フレームワーク不使用）。
- ES modules は使用していません。`file://` で直接開いたときにChrome/Edgeで
  CORSエラーになるためです。代わりに `window.WordTestApp` というグローバル名前空間に
  各モジュールをぶら下げる「名前空間パターン」を採用しています。
- `index.html` の `<script>` 読み込み順が依存関係の順序になっているため、
  ファイルを追加する場合は既存の順序を崩さないようにしてください。
- Excel解析ライブラリ（SheetJS）はCDNではなく `lib/` にオフラインでvendorしています。
- Excel読込・Googleスプレッドシート読込は、どちらも最終的に「行×列の文字列配列」に
  変換したうえで `wordRowParser.js` に渡す設計です（列判定・行の検証ロジックを
  二重実装しないため）。データ形式に手を入れる場合は `wordRowParser.js` のみを
  変更すれば、両方の読込方式に反映されます。

### 動作確認方法

このプロジェクトにはビルド・ユニットテストの仕組みは含まれていません。実装時は
Playwright（Chromium）を使い、実際のExcelサンプルを読み込んでの生成・プレビュー・
印刷レイアウト・設定の保存/復元を手動で検証しています。変更を加えた際は、同様に
実際のブラウザ操作（ファイル読込 → 条件設定 → 生成 → 印刷プレビュー）で
動作確認することを推奨します。

## 12. GitHub Pagesでの公開（管理者向け）

このセクションは、アプリをGitHub Pagesで公開・更新する担当者向けの手順です。
利用者（先生・生徒）は読む必要はありません。

### 採用している公開方式（Ver2.3）

アプリ本体（`index.html` / `css/` / `js/` / `lib/`）を**リポジトリ直下**に配置しています。
これにより、GitHub Pagesの標準機能（設定だけで使える「Deploy from a branch」、または
このリポジトリに用意済みのGitHub Actionsワークフロー）のどちらでも、
`https://<ユーザー名>.github.io/<リポジトリ名>/` を開くだけで直接アプリが起動します。

> **Ver1.3〜Ver2.2との違い**: 以前は本体一式を`src/`フォルダにまとめ、GitHub Actions経由
> でのみ公開する方式を採用していました（`src/`をそのまま動かせる利点がありました）。
> しかし、GitHub PagesのSource設定が「Deploy from a branch」（ブランチ直接配信、
> Jekyll処理あり）になっていると、リポジトリ直下に`index.html`が無いためJekyllが
> `README.md`を代わりにホームページとして表示してしまう問題が判明しました。
> この問題を確実に避けるため、Ver2.3で本体一式をリポジトリ直下へ移動しました。

#### 方法1: Deploy from a branch（最も簡単・追加設定不要）

1. GitHubのリポジトリ画面で **Settings → Pages** を開きます。
2. 「Build and deployment」の **Source** を **Deploy from a branch** に設定し、
   ブランチを `main` 、フォルダを `/(root)` に設定して **Save** します。
3. `main` ブランチへpushすると、数分後にPagesへ反映されます。

リポジトリ直下に`.nojekyll`（空ファイル）を配置済みのため、Jekyllによる余計な変換は
行われず、`index.html`がそのまま配信されます。

#### 方法2: GitHub Actions（`Data/`等を確実に公開対象から除外したい場合）

`.github/workflows/deploy-pages.yml` を使うと、`index.html` / `css/` / `js/` / `lib/`
だけを一時フォルダへコピーしてから公開します。`Data/`・`docs/`・`README.md`等は
このコピーに含まれないため、**リポジトリがPublicであっても、Pages経由では絶対に
配信されません**（後述の「注意事項」も参照）。

1. GitHubのリポジトリ画面で **Settings → Pages** を開きます。
2. 「Build and deployment」の **Source** を **GitHub Actions** に設定します。
3. `main` ブランチへpushすると、Actionsタブでワークフローが自動実行され、
   数分後にPagesへ公開されます。

どちらの方法も、初回のリポジトリ作成・push手順自体は共通です
（`git init` → `git add` → `git commit` → `git remote add origin ...` → `git push`。
※ デフォルトブランチ名が`main`でない場合は、方法2を使うときのみ
`.github/workflows/deploy-pages.yml`の`branches: ["main"]`を実際のブランチ名に
合わせて変更してください）。

### 公開URLの形式

```
https://<GitHubユーザー名>.github.io/<リポジトリ名>/
```

組織（Organization）で公開する場合は `<組織名>.github.io/<リポジトリ名>/` になります。
実際のURLは、Settings → Pages 画面、またはActionsのデプロイ結果に表示されます。

### 更新時の手順

`index.html` / `css/` / `js/` / `lib/` のいずれかを変更して `main` ブランチへpushする
だけです。方法1（Deploy from a branch）ならpush内容がそのまま反映され、方法2
（GitHub Actions）なら該当フォルダの変更を検知して自動的に再デプロイします
（`Data/`・`docs/`・`README.md`のみの変更では、方法2のワークフローは再デプロイを
行いません）。どちらの方法でも、手動でのビルド・コピー・デプロイ操作は不要です。

### 注意事項（公開範囲について）

- **方法2（GitHub Actions）を使う場合**: 公開されるのは`index.html`・`css/`・`js/`・
  `lib/`のみです。`Data/`フォルダ（単語データExcel）はステージング処理の対象外のため、
  Pages経由では配信されません（利用者は自分のExcel/Googleスプレッドシートを選択して
  読み込む方式のままです）。
- **方法1（Deploy from a branch）を使う場合**: リポジトリのブランチの中身がそのまま
  配信されるため、`Data/`がリポジトリに**コミットされていれば**Pages経由でも配信されて
  しまいます。このリポジトリでは`Data/`を`.gitignore`に登録済みで、通常の操作では
  コミットされない設定になっていますが、意図せず`git add -f`等で追加しないよう
  注意してください。
- いずれの方法でも、**リポジトリ自体をPublicで作成した場合**、`Data/`がコミットされて
  いれば、Pages経由で配信されなくてもGitHubのリポジトリ画面（コード閲覧・git履歴）
  からは誰でも閲覧できてしまう点は変わりません。`Data/`の単語データが教材・書籍等に
  基づく著作物である場合は、リポジトリをPrivateにする、または`Data/`を確実にコミット
  対象から外す運用を徹底してください（Privateリポジトリのページ公開にはGitHubの
  有料プランが必要な場合があります）。この判断は本ドキュメントでは行わず、公開前に
  方針を決めてください。
- 読み込んだExcelデータ・保存されるテスト条件は、これまでどおりブラウザの
  localStorage内のみで完結し、GitHub Pages上でも外部（GitHubを含む）へは
  一切送信されません。
